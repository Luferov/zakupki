# MVP план: НеМаркеплейс

## 1. Базовая платформа (выполнено)
- Next.js + App Router
- Интеграция MUI и единая тема
- Глобальный store (товары, корзина, уведомления)
- Роуты: `/`, `/buy`, `/sell`, `/cart`, `/checkout`

Критерий готовности:
- Приложение проходит `npm run lint` и `npm run build`

## 2. Каталог и покупка (выполнено)
- Поиск по названию
- Фильтры по категории и цене
- Карточки товаров: фото, цена, описание, остаток
- Добавление в корзину с обновлением счётчика
- Блок отзывов
- Модалка связи с продавцом (Email / Max / WhatsApp)

Критерий готовности:
- Пользователь может найти товар и добавить его в корзину

## 3. Публикация товара (выполнено)
- Форма продажи: фото, название, характеристики, цена, количество, категория
- Валидация полей
- Публикация в общий каталог (через store)

Критерий готовности:
- Новый товар появляется на странице покупки без перезапуска приложения

## 4. Корзина и checkout (выполнено)
- Корзина: список, удаление, итоговая сумма
- Checkout: способ оплаты, карта-заглушка, адрес, телефон
- Кнопка оплаты и очистка корзины после успешного оформления

Критерий готовности:
- Пользователь проходит полный путь от каталога до оплаты

## 5. Стабилизация (следующий этап)
- E2E тесты критического сценария (buy -> cart -> checkout)
- Улучшение валидаций (телефон, индекс, лимиты остатков)
- Обработка edge-cases (битые данные в localStorage)

Критерий готовности:
- Ключевые пользовательские сценарии покрыты автотестами

## 6. Backend и данные (по необходимости)
- Подключение Prisma и PostgreSQL
- Миграция данных из локального store в API
- Авторизация продавцов и история заказов

Критерий готовности:
- Данные сохраняются в БД и доступны после перезапуска приложения
